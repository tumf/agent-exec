# 変更提案: agent-exec v0.1 ジョブ保存と XDG パス

## 概要

ジョブ保存先の解決順序、ジョブディレクトリ構造、`meta.json`/`state.json` の必須フィールド、原子的な書き込み方針を固定する。

## 目的

- 実行結果の永続化と再取得を安定させる
- XDG ベースの保存先を標準化し、Windows を含むクロスプラットフォーム対応を明確化する

## スコープ

- ジョブ保存先の優先順位と既定パス
- ジョブディレクトリ構造
- `meta.json`/`state.json` の必須フィールド
- 原子的な書き込み（write temp + rename）

## スコープ外

- 実行監視の具体的な実装
- Windows のプロセスツリー制御

## 分割方針

外部契約・実行監視・Windows 実装とは分離する。
