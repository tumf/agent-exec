# 設計メモ: run の既定 snapshot-after を 10,000ms に変更

## 目的

- `agent-exec run` の既定待機時間を 10 秒に揃え、長時間実行のコマンドでも
  既定でスナップショットが返るようにする。

## 方針

- CLI の `run --snapshot-after` の既定値を 10,000ms に変更する。
- `snapshot-after=0` を明示した場合は従来どおり即時返却（snapshot 省略可）とする。

## 影響範囲

- `run` のデフォルト挙動のみ変更（フラグ指定時は変更なし）。
- 統合テストの待機時間に影響するため、必要に応じて `--snapshot-after 0` を明示する。
