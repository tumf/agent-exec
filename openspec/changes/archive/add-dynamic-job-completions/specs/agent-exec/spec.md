## ADDED Requirements

### Requirement: dynamic job-ID completion

Shell completion scripts generated by `agent-exec completions <SHELL>` MUST provide dynamic completion candidates for `<job_id>` arguments. The completer MUST enumerate existing job directories under the resolved root. If the root directory does not exist or is unreadable, the completer MUST return an empty candidate list (no error). Candidates SHOULD include a description annotation showing the job state when available.

#### Scenario: tab-complete job ID for status

**Given**: two jobs exist under the default root with IDs `01AAA...` and `01BBB...`
**When**: the user triggers completion for `agent-exec status <TAB>`
**Then**: both job IDs appear as completion candidates

#### Scenario: tab-complete job ID for kill filters to running

**Given**: job `01AAA...` is running and job `01BBB...` is exited
**When**: the user triggers completion for `agent-exec kill <TAB>`
**Then**: only `01AAA...` appears as a completion candidate

#### Scenario: empty root directory

**Given**: the jobs root directory does not exist
**When**: the user triggers completion for `agent-exec status <TAB>`
**Then**: no candidates are returned and no error occurs
