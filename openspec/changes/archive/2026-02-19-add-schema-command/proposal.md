# 変更提案: add-schema-command

## 背景/課題
stdout の JSON 契約が厳密である一方、外部統合向けに「公式な JSON Schema の取得手段」がない。SDK/他言語統合の初期実装や破壊的変更検知が難しい。

## 目的
- `agent-exec schema` で機械可読な JSON Schema を取得できるようにする
- 既存の CLI 契約を崩さずに、外部統合の効率を上げる

## スコープ
- `schema` サブコマンドの追加
- `schema` の JSON レスポンス定義
- 仕様・テストの更新

## 非スコープ
- Schema の自動生成ツールの導入
- 破壊的変更の自動リリース管理

## 成功指標
- TS/Python などでの型生成が容易になる
- 変更時に Schema 差分が検知できる

## 依存/リスク
- JSON Schema の表現範囲と実装の整合（静的ファイル管理でリスク低減）
