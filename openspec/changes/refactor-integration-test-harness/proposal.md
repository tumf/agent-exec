# 変更提案: 統合テストのテストハーネス整理

## 概要

統合テストの `tempdir` 作成と `AGENT_EXEC_ROOT` 設定、コマンド実行の重複を整理し、テストハーネスを導入する。テスト内容は変えない。

## 目的

- テストのボイラープレートを削減し可読性を上げる
- 追加テスト時のセットアップ漏れを防ぐ

## スコープ

- `tests/integration.rs` の共通セットアップ・コマンド実行のヘルパー化
- 既存アサーションの維持

## スコープ外

- 既存テストの仕様変更
- テストケースの削除や統合

## 受け入れ基準

- すべての統合テストが従来どおり成功する
- 既存のテスト意図（検証内容）が変わらない
- 新しいハーネスが明確な責務で再利用可能になっている
